# Отчёт по Спринту 6: Импорт, экспорт и безопасный обмен данными

**Период:** Основная работа велась 25–28 мая 2026 года (согласно коммитам).

**Цель:** Реализован безопасный импорт/экспорт хранилища в различных форматах, добавлена поддержка Bitwarden и LastPass, создана система безопасного обмена записями (шаринг) с использованием QR-кодов и публичных ключей.

Спринт 6. На шестом этапе реализуется безопасный обмен данными с:
- экспортом записей в зашифрованном JSON, CSV и Bitwarden JSON,
- импортом из Bitwarden JSON, LastPass CSV и нативного формата,
- автоматическим определением формата файла при импорте,
- шифрованием экспорта через AES-256-GCM с отдельным паролем,
- безопасным шарингом отдельных записей через пароль или публичный ключ (RSA),
- генерацией и сканированием QR-кодов для обмена ключами и пакетами шаринга,
- защитой от formula injection в CSV и санитизацией всех полей.

## Работа файлов

### Сервисы импорта/экспорта (`src/core/import_export/`)

- **`exporter.py`** — `VaultExporter`:
  - `export()` — основной метод экспорта с проверкой пароля
  - `_export_encrypted_json()` — экспорт в зашифрованный JSON (AES-256-GCM)
  - `_export_csv()` — экспорт в CSV с защитой от formula injection
  - `_export_bitwarden()` — экспорт в Bitwarden JSON (совместимый формат)
  - `_get_entries_for_export()` — получение записей с фильтрацией полей
  - `_record_history()` — логирование в историю импорта/экспорта

- **`importer.py`** — `VaultImporter`:
  - `import_file()` — основной метод импорта с таймаутом и лимитом файла
  - `_detect_format()` — авто-определение формата по содержимому
  - `_parse_encrypted_json()` — расшифровка нативного формата
  - `_parse_bitwarden()` — парсинг Bitwarden JSON
  - `_parse_lastpass_csv()` — парсинг LastPass CSV
  - `_parse_csv()` — универсальный CSV с авто-определением колонок
  - `_sanitize_entry()` — санитизация полей (защита от XSS и injection)

- **`sharing_service.py`** — `SharingService`:
  - `share_entry()` — создание зашифрованного пакета шаринга
  - `receive_entry()` — расшифровка полученного пакета
  - `_create_password_package()` — шифрование паролем (PBKDF2 + AES-GCM)
  - `_create_public_key_package()` — гибридное шифрование (RSA-OAEP + AES-GCM)
  - `generate_key_pair()` — генерация RSA-2048 ключей
  - `save_contact()` / `get_contacts()` — управление контактами
  - `revoke_share()` — отзыв активного шаринга

- **`key_exchange.py`** — `QRCodeService`:
  - `generate_qr_codes()` — генерация QR-кодов с chunking для больших данных
  - `generate_public_key_qr()` — QR для публичного ключа
  - `generate_share_qr()` — QR для пакета шаринга
  - `scan_qr_from_file()` — сканирование QR из изображения
  - `decode_chunks()` — сборка данных из нескольких QR-кодов
  - защита от replay-атак через timestamp + nonce + TTL

- **ARC-1, ARC-2, ARC-3 (Архитектура):** Создана директория `src/core/import_export/` с `exporter.py`, `importer.py`, `sharing_service.py`, `key_exchange.py`. Используются отдельные ключи от мастер-ключа.

- **EXP-1, EXP-2, EXP-3, EXP-4 (Экспорт):** Реализованы форматы encrypted_json, CSV, Bitwarden JSON. Поддерживается AES-256-GCM шифрование, выбор записей, исключение полей, сжатие GZIP. Требуется подтверждение мастер-пароля.

- **IMP-1, IMP-2, IMP-3, IMP-4 (Импорт):** Поддержка encrypted_json, CSV, Bitwarden JSON, LastPass CSV. Реализована санитизация полей, проверка на вредоносные паттерны, лимит файла 10 МБ, таймаут 30 секунд. Режимы: merge, replace, dry_run.

- **SHR-1, SHR-2, SHR-3, SHR-4 (Шаринг):** Поддержка пароля и публичного ключа. Пакет включает метаданные (отправитель, срок действия, права). Workflow отправителя и получателя полностью реализован.

- **QR-1, QR-2, QR-3, QR-4 (QR-коды):** Генерация QR для публичных ключей и пакетов шаринга. Сканирование из файлов. Защита от replay-атак через TTL (5 минут по умолчанию) и nonce.

- **CRY-1, CRY-2, CRY-3, CRY-4 (Криптография):** PBKDF2-HMAC-SHA256 (100,000 итераций) для пароля. RSA-OAEP + AES-256-GCM для публичного ключа. Forward secrecy через эфемерные ключи. Целостность через HMAC и цифровые подписи.

- **DB-1, DB-2, DB-3 (База данных):** Добавлены таблицы `shared_entries`, `import_export_history`, `contacts` с индексами.

- **FMT-1, FMT-2, FMT-3 (Форматы файлов):** Спецификации соответствуют ТЗ. Поддерживается верификация через SHA-256 хэш и подпись.

- **TEST-1, TEST-2, TEST-3, TEST-4, TEST-5 (Тестирование):** Написаны тесты round-trip, interoperability, sharing security, QR code, performance.

- **SEC-1, SEC-2, SEC-3, SEC-4, SEC-5 (Безопасность):** Экспорт по умолчанию шифрованный (кроме миграции). Валидация и санитизация всех полей. Разделение ключей. Очистка временных ключей из памяти. Проверка на вредоносные паттерны.

## Ключевые технические решения

1. **Авто-определение формата:** по содержимому файла (первые 4 КБ) и расширению
2. **Защита от formula injection:** добавление апострофа перед символами `=+-@` в CSV
3. **Гибридное шифрование:** RSA-OAEP для ключа + AES-GCM для данных
4. **QR chunking:** разбиение больших данных на несколько QR-кодов с контрольными суммами
5. **Хэш целостности:** SHA-256 для верификации импортируемых файлов
6. **Асинхронные операции:** фоновые потоки для долгих импортов/экспортов

## Основные проблемы и решения

- **Пароль требовался для всех форматов экспорта** → исправлено: только для encrypted_json
- **NoneType ошибка при экспорте в Bitwarden** → добавлена проверка `if format == "encrypted_json"`
- **Bitwarden не принимал JSON** → исправлена структура: добавлены `reprompt`, `fields`, `fido2Credentials`, `passwordHistory`, `creationDate`, `revisionDate`
- **Файлы >10 МБ блокировали приложение** → добавлена проверка размера и таймаут 30 секунд
- **CSV formula injection** → добавлен апостроф перед потенциально опасными символами
- **Отсутствие кэша при поиске** → добавлено кэширование расшифрованных записей
