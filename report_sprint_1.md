# СПРИНТ 1

Это отчёт о том, что сделано в первом спринте

Проект — менеджер паролей с локальной sqlite‑базой и GUI. Пока есть заглушки (шифрование, аудит), но структура готова из ТЗ: конфиг, мастер настройки, пул БД, миграции, шина событий и базовые диалоги.

На первом этапе (Спринт 1) создается безопасный, расширяемый фундамент с:
- зашифрованной базой данных,
- модульной архитектурой,
- базовой оболочкой графического интерфейса.

Когда пользователь запускает программу: если БД нет — мастер настройки попросит пароль и путь к файлу, затем программа создаст таблицы и откроет главное окно. Там можно добавлять записи, пароли сохраняются зашифрованными (пока заглушка). События от операций идут в EventBus, а тестовый AuditLogger пишет строки в лог.

## Работа файлов

- `src/app.py` — запуск приложения: загрузка конфига, проверка БД, запуск мастера если нужно, создание пула БД и миграций, показ главного окна.

- `src/core/config.py` — чтение/запись `config.json`, хранение пути к БД и настроек, генерация ключа (сохранение соли).

- `src/core/key_manager.py` — генерация ключа из пароля (заглушка), очистка буфера в памяти.

- `src/core/state_manager.py` — состояние сессии, буфер обмена, таймеры авто‑блокировки.

- `src/core/events.py` — простой EventBus: подписка, отписка, публикация; `AuditLogger`‑заглушка пишет в файл.

- `src/core/crypto/*` — интерфейс шифрования и заглушка `AES256Placeholder` (XOR) для тестов.

- `src/database/db.py` — `DatabasePool`: пул sqlite‑соединений, миграции (создание таблиц и индексов).

- `src/database/models.py` — `VaultEntries`, `AuditLog`, `Settings`, `KeyStore` — репозитории для БД.

- GUI (`src/gui/`) — `SetupWizard`, `MainWindow`, `SettingsDialog`, виджеты (`PasswordEntry`, `SecureTable`, `AuditLogViewer`).

- `tests/` — unit и интеграционные тесты; в `conftest.py` — фикстуры `test_db` и `qapp`.

## Работа программы для пользователя

1. Запускает программу.
2. Если БД нет — мастер попросит мастер‑пароль и файл БД.
3. Программа создаст таблицы и откроет главное окно.
4. Можно добавить запись: пароль шифруется и сохраняется.
5. События публикуются, AuditLogger получает их и пишет строки в лог.
