# Отчёт по Спринту 3: CRUD-операции и AES-GCM шифрование

**Период:** Основная работа велась 31 марта – 3 апреля 2026 года.

**Цель:** Реализован полный CRUD для записей хранилища, заменена XOR-заглушка на AES-256-GCM, добавлен генератор паролей и улучшен табличный интерфейс.

Спринт 3. На третьем этапе реализуется полноценное хранилище учетных данных с:
- аутентифицированным шифрованием AES-256-GCM для каждой записи,
- уникальным nonce для каждой операции шифрования,
- генератором криптостойких паролей через secrets,
- табличным интерфейсом с сортировкой, поиском и контекстным меню,
- CRUD-операциями с публикацией событий и мягким удалением.

## Выполненные требования

* **ARC-1, ARC-2:** Создана `src/core/vault/` с `entry_manager.py`, `encryption_service.py`, `password_generator.py`.

* **ENC-1 - ENC-5:** AES-256-GCM с 12-байтовым nonce.

  * Формат: `nonce + ciphertext + tag`

* **DATA-1 - DATA-2:** Модель записи содержит `title`, `username`, `password`, `url`, `notes`, `category`, `version`.

* **CRUD-1 - CRUD-4:** `EntryManager` реализует создание, чтение, обновление, удаление + soft delete (TTL 30 дней).

* **GEN-1 - GEN-4:** `PasswordGenerator` с `secrets.choice`, гарантией символов и историей 20 паролей.

* **GUI-1 - GUI-4:** `SecureTable` с сортировкой, контекстным меню, копированием, редактированием.

## Ключевые технические решения

1. Индивидуальное создание AESGCM на каждую операцию.
2. Blob-структура `nonce + ciphertext + tag`.
3. Secure wipe после расшифровки данных.
