# Руководство пользователя

---

## 1. Введение

**CryptoSafe Manager** — это кроссплатформенный менеджер паролей с локальной зашифрованной базой данных и графическим интерфейсом. Все данные хранятся исключительно на вашем устройстве и никогда не передаются в облако.

### Основные возможности

- **Шифрование AES-256-GCM** — каждая запись шифруется индивидуальным nonce; подмена данных обнаруживается автоматически.
- **Argon2id + PBKDF2** — защита мастер-пароля от брутфорса с памятью 64 МБ и тремя итерациями.
- **Безопасный буфер обмена** — автоматическая очистка через настраиваемый таймаут, блокировка памяти (mlock/VirtualLock), предупреждение за 5 секунд.
- **Импорт и экспорт** — зашифрованный JSON, CSV, Bitwarden JSON, LastPass CSV.
- **Журнал аудита** — криптографические подписи Ed25519, hash chain, экспорт в JSON/CSV/PDF.
- **Режим паники** — мгновенная очистка всех чувствительных данных по горячей клавише.
- **Генератор паролей** — криптостойкие пароли через `secrets`, без повторений в истории.

> **Примечание о конфиденциальности.** Пароли и другие чувствительные данные никогда не попадают в журнал аудита. В логах они заменяются на `[REDACTED]`.

## 2. Установка и запуск

### Системные требования

- Python 3.12 или выше
- pip
- Git
- virtualenv / venv
- SQLite3 (входит в стандартную библиотеку Python)

### Установка из исходников

```bash
# 1. Клонировать репозиторий
git clone https://github.com/your-username/cryptosafe-manager.git
cd cryptosafe-manager

# 2. Создать виртуальное окружение
python -m venv venv

# 3. Активировать окружение
# Linux / macOS:
source venv/bin/activate
# Windows PowerShell:
venv\Scripts\activate

# 4. Обновить pip и установить зависимости
python -m pip install --upgrade pip
pip install -r requirements.txt

# 5. Запустить приложение
python src/app.py
```

### Опциональные зависимости

Для работы с QR-кодами (обмен ключами и пакетами шаринга) установите дополнительные пакеты:

```bash
pip install qrcode[pil] pyzbar Pillow
```

### Создание исполняемого файла (PyInstaller)

Если вы хотите запускать приложение без Python:

```bash
pip install pyinstaller
pyinstaller --onefile --windowed src/app.py --name CryptoSafeManager
```

Готовый исполняемый файл появится в папке `dist/`.

## 3. Первый запуск и настройка хранилища

При первом запуске откроется **Мастер первоначальной настройки**.

### Шаг 1: Выбор файла базы данных

Нажмите **Выбрать...** и укажите путь, где будет храниться зашифрованная база данных (файл `.db`). Выберите место, доступное только вам — например, домашнюю папку или зашифрованный раздел диска.

> **[Скриншот: диалог Мастера настройки с полем выбора файла БД]**

### Шаг 2: Создание мастер-пароля

Мастер-пароль защищает всё хранилище. Он **никогда не сохраняется на диск** — только его производный хэш Argon2id.

**Требования к мастер-паролю:**
- Минимум 12 символов
- Заглавные буквы (A–Z)
- Строчные буквы (a–z)
- Цифры (0–9)
- Специальные символы (!@#$%^&* и другие)
- Пароль не должен быть из списка самых распространённых (например, `password123`)

> **⚠️ Внимание!** Мастер-пароль невозможно восстановить. Если вы его забудете — доступ к данным будет утерян навсегда. Сохраните пароль в надёжном месте.

### Шаг 3: Завершение настройки

Нажмите **Готово**. Приложение создаст базу данных, сгенерирует криптографические ключи и откроет главное окно.

## 4. Основная работа с записями

> **[Скриншот: главное окно приложения с таблицей записей]**

### Добавление записи

1. В меню **Правка** выберите **Добавить** (или нажмите соответствующую кнопку на панели).
2. Заполните поля: Название, Логин, Пароль, URL, Заметки.
3. Нажмите **OK** — запись зашифруется и сохранится в базе.

> **Совет.** Воспользуйтесь кнопкой **🔄 Сгенерировать**, чтобы создать надёжный пароль автоматически. Генератор создаёт пароли длиной 16 символов из заглавных, строчных букв, цифр и спецсимволов, без визуально похожих символов (l, I, O, 0).

### Редактирование записи

Дважды щёлкните по записи в таблице или выберите её и нажмите **Правка → Изменить**. После изменений нажмите **OK** — запись перешифруется с новыми данными.

### Удаление записи

Выберите запись и нажмите **Правка → Удалить**. Подтвердите удаление в диалоге. По умолчанию запись помещается в «корзину» на 30 дней (мягкое удаление), после чего удаляется окончательно.

### Поиск и фильтрация

Введите текст в строку поиска над таблицей. Поддерживаются:

- **Полнотекстовый поиск** — по всем полям записи (Название, Логин, URL, Заметки).
- **Нечёткий поиск** — небольшие опечатки допустимы (например, `Gogle` найдёт `Google`).
- **Фильтр по полю** — используйте синтаксис `поле:значение`. Например: `title:работа`, `username:admin`, `url:github`.

### Просмотр паролей

По умолчанию пароли в таблице скрыты (`••••••••`). Чтобы их показать:
- Нажмите **Ctrl+Shift+P** или отметьте пункт **Вид → Показать пароли**.
- Для отдельной записи: щёлкните правой кнопкой → **👁️ Показать пароль** (пароль отобразится на 30 секунд, после чего окно закроется автоматически).

## 5. Безопасный буфер обмена

### Копирование данных

Щёлкните правой кнопкой на записи в таблице. Доступные варианты:
- **📋 Копировать пароль**
- **📋 Копировать логин**
- **📋 Копировать URL**

После копирования в строке состояния внизу окна появится отсчёт: `📋 Буфер [password]: 30с`.

> **[Скриншот: строка состояния с активным таймером буфера обмена]**

### Автоматическая очистка

Буфер обмена очищается автоматически по истечении таймаута (по умолчанию 30 секунд). За 5 секунд до очистки появится предупреждение: `⚠️ Буфер обмена очистится через 5 секунд`.

### Настройка таймаута

Откройте **Вид → Настройки → Безопасность**. Доступны готовые профили:
- **Стандарт** — 30 секунд
- **Безопасный** — 15 секунд
- **Публичный ПК** — 5 секунд

Или задайте произвольное значение от 5 до 300 секунд.

### Индикация в трее

Иконка в системном трее показывает подсказку: `📋 password — очистка через 28с`.

### Блокировка при подозрительном доступе

Если другое приложение изменит содержимое буфера обмена, CryptoSafe немедленно его очистит. После 3 таких событий копирование будет заблокировано. Чтобы разблокировать, выберите **Безопасность → Разблокировать буфер обмена**.

### Предпросмотр буфера

Меню **Безопасность → Предпросмотр буфера обмена** покажет тип данных, источник и время до очистки. Для просмотра содержимого потребуется ввести мастер-пароль.

## 6. Импорт и экспорт

### Экспорт хранилища

Откройте **Файл → Экспорт** (или соответствующий пункт меню).

| Формат | Шифрование | Назначение |
|--------|-----------|------------|
| 🔒 CryptoSafe JSON | AES-256-GCM | Резервные копии, перенос между устройствами |
| 📄 CSV | Нет | Миграция на другой менеджер паролей |
| 🔑 Bitwarden JSON | Нет | Переход на Bitwarden |

> **⚠️ Внимание!** Форматы CSV и Bitwarden JSON сохраняют пароли в открытом виде. Храните эти файлы в безопасном месте и удаляйте после использования.

**Параметры экспорта:**
- Выбор конкретных записей (или всех сразу)
- Исключение заметок и/или тегов
- Сжатие GZIP (уменьшает размер файла на 50–70%)

### Импорт записей

Откройте **Файл → Импорт**. Приложение автоматически определит формат файла.

Поддерживаемые форматы: CryptoSafe JSON, CSV, Bitwarden JSON, LastPass CSV.

**Режимы импорта:**
- **Объединить (merge)** — добавить новые, обновить совпадающие записи.
- **Заменить (replace)** — удалить все текущие записи и импортировать заново. ⚠️ Необратимо!
- **Превью (dry run)** — только показать, что будет импортировано, без сохранения.

## 7. Журнал аудита

Откройте **Вид → Логи**.

> **[Скриншот: окно просмотра журнала аудита с фильтрами]**

### Просмотр событий

В журнале фиксируются все важные действия:
- Входы и выходы (успешные и неудачные)
- Создание, изменение, удаление записей
- Операции с буфером обмена
- Изменения настроек
- Системные события (запуск, блокировка)

### Фильтрация

Используйте панель фильтров:
- **Поиск** — полнотекстовый поиск по деталям события.
- **Тип события** — выпадающий список с типами (LOGIN_SUCCESS, ENTRY_CREATED и др.).
- **Severity** — INFO, WARN, ERROR, CRITICAL.
- **Дата от/до** — диапазон дат.

### Проверка целостности логов

Каждая запись журнала подписана алгоритмом Ed25519 и связана с предыдущей через hash chain. Это делает невозможным незаметное изменение или удаление записей.

Чтобы запустить проверку вручную: **Вид → Проверить целостность логов**. Результат отобразится в виде детального отчёта, который можно сохранить в JSON.

При запуске приложения автоматически проверяются последние 1000 записей. При обнаружении нарушений появится предупреждение.

### Экспорт журнала

В окне журнала нажмите кнопку экспорта и выберите формат: JSON, CSV или PDF. Для подтверждения потребуется ввести мастер-пароль.

## 8. Смена мастер-пароля

Откройте **Безопасность → Сменить мастер-пароль**.

**Процесс смены:**
1. Введите текущий пароль (проверка через Argon2id).
2. Введите новый пароль и его подтверждение.
3. Новый пароль проходит проверку надёжности.
4. Все записи в хранилище перешифровываются с новым ключом в фоновом потоке — GUI не зависает.
5. Индикатор прогресса показывает ход операции.

> **Совет.** Если смена пароля завершится с ошибкой — транзакция автоматически откатится. Старый пароль продолжит работать, данные не будут повреждены.

## 9. Режим паники (Panic Mode)

**Горячая клавиша: Ctrl+Shift+1**

Режим паники предназначен для ситуаций, когда нужно немедленно скрыть все чувствительные данные.

**Что происходит при активации:**
- Буфер обмена немедленно очищается.
- Хранилище блокируется (ключи удаляются из памяти).
- Все окна приложения закрываются.
- Чувствительные данные в памяти перезаписываются нулями.

**Скрытные действия (stealth mode, настраивается в конфигурации):**
- Отображение фальшивого сообщения об ошибке системы.
- Запуск приложения-приманки вместо CryptoSafe.

После активации паники для доступа к хранилищу потребуется повторный ввод мастер-пароля.

## 11. Устранение неполадок

### Забытый мастер-пароль

> **⚠️ Мастер-пароль невозможно восстановить.** Данные хранилища зашифрованы и без пароля недоступны ни вам, ни разработчикам.

Единственный выход — создать новое хранилище и, если возможно, восстановить данные из незашифрованной резервной копии.

### Восстановление из резервной копии

1. Закройте приложение.
2. Замените файл `.db` резервной копией.
3. Запустите приложение и введите мастер-пароль, актуальный на момент создания резервной копии.

### Ошибка «Vault is locked»

Хранилище заблокировано. Это происходит после ввода неверного пароля, истечения таймера неактивности или блокировки при сворачивании окна. Введите мастер-пароль в диалоге входа.

### Ошибка «Copying is blocked» (Буфер заблокирован)

Обнаружен подозрительный доступ к буферу обмена. Откройте **Безопасность → Разблокировать буфер обмена**.

### Предупреждение о нарушении целостности аудит-лога

Если при запуске появилось сообщение о нарушении целостности журнала, не игнорируйте его. Сделайте резервную копию базы данных и выполните полную проверку через **Вид → Проверить целостность логов**.

### Приложение не запускается: ошибка импорта модуля

Убедитесь, что виртуальное окружение активировано и зависимости установлены:

```bash
source venv/bin/activate   # Linux/macOS
pip install -r requirements.txt
python src/app.py
```
